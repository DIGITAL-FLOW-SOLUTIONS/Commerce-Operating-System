🔵 ARTIFACT 5 — STORES SYSTEM (/stores)
“The Commerce Engine”
🧠 🎯 PRIMARY PURPOSE

The Stores System is:

A multi-tenant storefront rendering engine that allows each merchant to launch and operate a fully functional e-commerce store based on selected store types (TikTok, Social, Boutique).

⚠️ CRITICAL DESIGN TRUTH

You are NOT building “themes”.

You are building:

3 Different Commerce Experiences (Products)

Each with:

- Different UX
- Different complexity
- Different user behavior

🧩 STORE TYPES (SYSTEM DEFINITIONS)
🟣 1. TIKTOK LIVE STORE (FAST COMMERCE)
🎯 Purpose:

Ultra-fast conversion during live selling

🧠 Characteristics:
- No browsing
- No distractions
- One goal: BUY NOW

📄 Pages Required:
1. Product Page (Homepage)
2. Checkout Page
3. Success Page
4. Error Page
5. Customer Tracking Page (Optional login)

🧱 PAGE BREAKDOWN
🔴 PRODUCT PAGE (HOME)
Content:
- Product image (large)
- Product name
- Price (very visible)
- “Buy Now” button

⚙️ Behavior:
- Single product focus OR small list
- Optimized for mobile scrolling
- Instant CTA

🔵 CHECKOUT PAGE
Fields:
- Phone number (REQUIRED)
- Quantity

⚙️ Behavior:
- Trigger payment (M-Pesa flow later)
- Minimal friction

🟢 SUCCESS PAGE
Content:
- “Payment Successful 🎉”
- Delivery form

Delivery Form Fields:
- Country
- County
- Town
- Email
- Phone (auto-filled)
- Additional notes

⚙️ Behavior:
- Save delivery info
- Create customer record
- Link to order

🔴 ERROR PAGE
- Payment failed
- Retry button

🟡 CUSTOMER TRACKING SYSTEM
Flow:
- User enters phone number
- System checks orders
- Optional password creation

⚙️ Behavior:
- Password optional
- OTP-based or simple verification

🟡 4. STORE ENGINE (MULTI-TENANT CORE)
🎯 Purpose:

Power all stores dynamically

📦 Model: Store
Fields:
- id
- tenant_id
- name
- type (tiktok, social, boutique)
- theme_id
- status
- domain

📦 Model: StoreTheme
- id
- type
- layout_config
- default_styles

⚙️ Behavior:

When merchant selects template:

1. Create store
2. Assign type
3. Import data (from scraping or manual)
4. Attach theme
5. Generate URL:
/stores/{id}/{store-name}

🤖 INSTRUCTIONS — STORE ENGINE
Design a multi-tenant store engine:

- Create store based on type (tiktok, social, boutique)
- Assign theme and layout
- Import merchant data (products, name, branding)
- Generate store URL
- Ensure tenant isolation

Support dynamic rendering per store.
